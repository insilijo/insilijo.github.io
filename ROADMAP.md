# insilijo.github.io Roadmap

## Current State (2026-03-23)

### Working well
- Site is built on a mature upstream theme with an established publishing workflow.
- Recent work suggests active content updates and project writeups.

### Quality assessment
- Overall quality: stable site foundation, but content/portfolio curation is the real work rather than engineering.
- The repo inherits a large amount of upstream theme surface area that is not the real product.
- Risk is less about code correctness and more about content drift, placeholder pages, and inconsistent project storytelling.

### Highest risks
- Upstream theme complexity can distract from the actual goal of maintaining a coherent personal/research site.
- Placeholder content and inconsistent project pages can make the portfolio feel unfinished even when the site builds correctly.
- Repo size and theme noise make it harder to see which changes are truly project-specific.

## Priority Roadmap

### Phase 1 — Content Hygiene
- [ ] Remove or replace placeholder pages and draft content.
- [ ] Standardize project pages so active work has consistent summaries, links, and status.
- [ ] Add/update writeups for current flagship projects.

### Phase 2 — Portfolio Narrative
- [ ] Make the site clearly communicate the relationship among GrAndMA, GeMMA, GIZMO, SQuID-INC, and scATMAN.
- [ ] Add a concise research/program overview page rather than relying on scattered posts.
- [ ] Ensure project maturity/status is obvious to visitors.

### Phase 3 — Theme Debt Reduction
- [ ] Minimize unused upstream theme features where practical.
- [ ] Document local customizations so future updates do not break them.

## Recommendation

This is a content-management roadmap, not a systems-engineering roadmap. The main goal should be a coherent public narrative of the current projects, not more site machinery.
